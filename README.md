# 🥄 Spoon (스푼)
> **"맛있는 정보를 한 스푼, 거래를 한 스푼" - C2C 기반 맛집 & 푸드 마켓플레이스**

![Spring Boot](https://img.shields.io/badge/SpringBoot-3.x-green) ![JSP](https://img.shields.io/badge/JSP-Jakarta-blue) ![Oracle](https://img.shields.io/badge/Database-Oracle-red) ![MyBatis](https://img.shields.io/badge/ORM-MyBatis-red)

## 📖 프로젝트 소개
Spoon은 단순한 정보 공유를 넘어, 사용자가 직접 맛집의 음식을 판매하거나 구매할 수 있는 **C2C(Customer to Customer) 플랫폼**입니다. 한 계정으로 구매와 판매를 동시에 할 수 있는 유연한 사용자 환경을 목표로 설계되었습니다.

---

## ✨ 핵심 구현 기능
### 1️⃣ 통합 멀티롤(Multi-Role) 시스템
- **구매자(Buyer) & 판매자(Seller) 전환**: 사용자는 별도의 가입 없이 본인 인증 및 판매 정보 등록만으로 판매자 권한을 획득할 수 있습니다.
- **역할 기반 인터페이스**: 동일 계정 내에서 일반 회원 기능과 판매자 관리 기능을 유기적으로 이용 가능합니다.

### 2️⃣ 📍 지능형 주소 검색 시스템
- **도로명주소 Open API 연동**: 검색 팝업을 통해 정확한 배송지 및 판매처 위치 데이터를 수집합니다.
- **API 라이프사이클 관리**: 승인키 만료 등 외부 API 장애 상황에 대한 즉각적인 대응 프로세스를 구축했습니다.

### 3️⃣ 🛒 마켓플레이스 및 게시판
- **상품 등록 및 관리**: MyBatis를 활용하여 상품 정보(이미지, 옵션, 가격 등)의 CRUD를 구현했습니다.
- **커뮤니티(Board)**: 사용자 간 정보 공유를 위한 자유게시판 기능을 제공합니다.

## 🏗️ 기술적 도전 및 해결 (Trouble Shooting)
### 🔄 Oracle Trigger를 이용한 데이터 동기화
- **도전**: 판매자(`Seller`)의 신뢰도(매너 온도 등)가 변경될 때 회원(`Member`) 정보에 실시간으로 반영되어야 했습니다.
- **해결**: DB 단에서 **Trigger**를 설계하여 판매자 정보 업데이트 시 회원 테이블의 데이터가 자동으로 동기화되도록 구현하여 데이터 무결성을 확보했습니다.

### 🗄️ 복합 관계 테이블 설계 및 쿼리 최적화
- **도전**: 회원-판매자-상품-주문으로 이어지는 복잡한 관계에서 데이터 삽입 시 `ORA-00913` 등의 데이터 불일치 이슈가 발생했습니다.
- **해결**: MyBatis 매퍼에서 컬럼명을 명시적으로 선언하고 DTO-DB 간 매핑 구조를 재정의하여 안정적인 데이터 처리를 실현했습니다.
